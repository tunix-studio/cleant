# Security Policy

## Reporting a vulnerability

Because tclean deletes files and runs with broad filesystem access, security
reports are taken seriously. **Please do not open a public issue for security
vulnerabilities.**

Instead, please report privately through GitHub:

- use GitHub’s **“Report a vulnerability”** (Security → Advisories) on this repository, or
- get in touch via my GitHub profile: **[@tunahancelik](https://github.com/tunahancelik)**.

Include: affected version, a description, reproduction steps, and the potential
impact (e.g. unintended file removal, path‑guard bypass, privilege issues).

We aim to acknowledge reports within a few days and to ship a fix as quickly as the
issue’s severity warrants. Responsible disclosure is appreciated — please give us a
reasonable window to release a fix before any public disclosure.

## Supported versions

The latest released version receives security fixes. Pre‑1.0, only the most recent
release is supported.
